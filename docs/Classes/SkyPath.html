  • start(apiKey:airline:userId:env:completion:)

    Start SDK. It will not track and provide data until started. Should be called when SkyPath service needs to start (for example on app start) only once until app is relaunched or after it was explicitly stopped by SkyPath.stop().

    Declaration

    Swift

    public func start(apiKey: String,
                  airline: String,
                  userId: String,
                  env: Environment? = nil,
                  completion: @escaping (_ error: StartError?) -> Void)

    Parameters

    apiKey

    The API key provided to you by SkyPath.
    airline

    The company ICAO code registered in SkyPath system.
    userId

    Any user identifier unique per company. Does not need to be registered in the SkyPath system.
    env

    Set it when SkyPath server is not whitelisted and a proxy server should be used, or when need dev env. Optional. Default is production.
    completion

    Will be called asynchronously on the main thread. error will have details in case SDK can’t start. If all is good SkyPathDelegate.didUpdateRecordingStatus(to:) will be called with recording: true.

  • dataQuery

  • dataUpdateFrequency

    Configure the frequency of data update from the server. For example, to save network traffic, you can set it to none to disable periodic new data fetch from the server at all, or set to minimal to update with bigger time intervals. Can be updated at any time in the project. Value is stored across app launches until changed. Default is fast.

    Declaration

    Swift

    public var dataUpdateFrequency: DataUpdateFrequency { get set }
  • dataHistoryTime

    Set history time to receive data for in an initial data request made on start. When started, SDK fetches initial data for time back in time and then receive only updates since last update. So if you don’t need for example 6 hours of data history, set this variable to number of hours that best suites your needs and save network traffic. Default is twoHours.

    Declaration

    Swift

    public var dataHistoryTime: DataHistoryTime { get set }

  • fetchData(refresh:)

    In case you need to get new data immediately (for example when opened the app) call this method and the result will be notified in corresponding delegate methods. Data is automatically fetched from the server by SDK out of the box, as well on changing data in dataQuery or dataUpdateFrequency. Use this function to let SDK fetch new data from the server immediately. Do not call it frequently. A recommended usage is to call it when the pilot taps a refresh button if it is such. The fetched data result will be notified in corresponding delegate methods.

    Declaration

    Swift

    public func fetchData(refresh: Bool = false)

    Parameters

    refresh

    If true, all data per query will be fetched.
