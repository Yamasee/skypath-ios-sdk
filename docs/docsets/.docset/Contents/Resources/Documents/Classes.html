  • YamaseeCore

    A wrapper class handling all communications between host app and internal Yamasee logic

    Overview

    YamaseeCore instantiates as a singleton object. Get the shared instance by calling YamaseeCore.shared

    Assign a custom object to the delegate property, conforming to the YamaseeCoreDelegate protocol. Assign the delegate before starting any YamaseeCore services. All turbulence, traffic and weather reports updates events are delivered to that delegate.

    Events only notify the delegate that a new data arrived from the server. It is the host app responsibility to fetch the updated data by calling the relevant function when the host app needs to redraw the information as Yamasee SDK doesn’t hold the state of the UI i.e. min/max altitude of the display, timespan for turbulence etc.

    YamaseeCore handles all communications with Yamasee servers. The host app should provide a baseURL that act as a proxy and forward all communication to Yamasee servers.

    The host app needs to initiate Yamasee server communications by logining in with partner login method and provided by Yamasee credentials. After a successful login the user token is saved for future app operations until logout is called or the user license is removed on the server.
